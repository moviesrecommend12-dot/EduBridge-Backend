عقد الربط مع تطبيق Flutter

هذه الصفحة تلخص ما يحتاجه تطبيق Flutter لتسجيل الجهاز واستقبال إشعارات Orbiet.

تسجيل جهاز Push /api/notifications/devices/
تفضيلات المستخدم /api/notifications/preferences/
إشعارات المستخدم /api/notifications/
تفاصيل إشعار /api/notifications/<id>/
سجل Push للمستخدم /api/notifications/deliveries/
المصادقة JWT Bearer token للمستخدم الحالي
حقل تسجيل الجهاز المعنى مثال
token FCM registration token من Firebase Messaging fcm-token-from-device
platform منصة الجهاز ANDROID / IOS / WEB
app_name اسم تطبيق Flutter orbiet_flutter
device_id معرف محلي للجهاز إن وجد android-id-or-installation-id
حقل Delivery API المعنى
notification_title عنوان الإشعار المرتبط بمحاولة الإرسال
notification_type نوع الإشعار المرتبط بمحاولة الإرسال
notification_deep_link الرابط الداخلي المرتبط بمحاولة الإرسال
device_platform منصة الجهاز
device_app_name اسم التطبيق المسجل للجهاز
device_identifier معرف الجهاز المرسل من Flutter إن وجد
device_is_active هل الجهاز لا يزال مفعّلًا لاستقبال Push
مفتاح Push payload المعنى
notification_id رقم الإشعار داخل Orbiet
notification_type GENERAL / GRADE / ATTENDANCE / ASSIGNMENT / INVOICE / PAYMENT / MESSAGE
related_object_type اسم الكائن المرتبط، مثل FeeInvoice أو Assignment
related_object_id رقم الكائن المرتبط
deep_link الرابط الداخلي الذي يجب أن يفتحه Flutter عند الضغط على الإشعار
Deep link الصفحة المقترحة في Flutter
orbiet://invoices/12 تفاصيل الفاتورة
orbiet://grades/8 تفاصيل الدرجة
orbiet://assignments/5 تفاصيل الواجب
orbiet://attendance?student_id=3 سجل حضور الطالب
orbiet://chat/rooms/2 غرفة المحادثة
